# Word

## < 소프트웨어 > 
- 소프트웨어: 컴퓨터나 스마트폰 같은 하드웨어 장치에 무엇을 해야 할지 지시하는 일련의 명령어 및 프로그램들의 집합

- 시스템 소프트웨어(System Software)
  - 하드웨어를 직접 제어하고 관리 -> 사용자가 컴퓨터를 켤 수 있게 해주는 핵심 기반(ex. 윈도우(Windows),맥OS(macOS),리눅스(Linux))

- 응용 소프트웨어(Application SoftWare)
  - 사용자가 특정 목적이나 작업을 수행하기 위해 사용하는 프로그램(ex. 웹 브라우저(크롬), 문서 작성 프로그램(한글,MS 워드), 게임 , 모바일 앱등)

#### - 특징

 - 하드웨어와 달리 물리적 실체가 없는 무형성, 끊임 없이 수정/발전 가능한 변경성/ 시간이 지나도 닳지 않는    비마모성을 가짐

----

## < SDLC >
- SDLC: 고품질 소프트웨어 -> 비용 측면에서 효율적 / 신속한 개발 
        -> 기획 ~ 유지보수까지 체계화한 프로세스

- 기획 및 분석 : 프로젝트의 목표, 범위 및 요구 사항 정의 
 
- 설계 : 시스템 아키텍처, UI/UX, 데이터베이스 및 보안을 계획
    
- 개발(코딩) : 설계 문서를 바탕으로 실제 프로그래밍 수행

- 테스트 : 버그나 오류를 찾아내고 요구사항 충족 여부를 검증

- 배포 : 완성된 소프트웨어를 사용자가 이용할 수 있는 환경에 릴리스 함

- 유지 보수 : 시스템 운영중 발생하는 문제를 해결 & 기능 업데이트

----

## < 해시 코드(Hash code) >
- 해시 코드(Hash code): 다양한 길이의 데이터를 "해시 함수"라는 수학적 알고리즘을 통하여 
                      고전된 길이의 고유한 숫자 값으로 변환한 결과물

- 특징: 1) 데이터 전체 비교(X) -> 해시 코드(정수 인덱스)를 통하여 위치 바로 찾음
          -> 데이터 베이스,자료구조에서 매우 빠름
        2) 해시 테이블에서 데이터를 배열에 저장하고 찾을때, 문자열/ 객체를 직접 인덱스로 사용             하는 대신 => 해시코드를 배열의 인덱스로 변환하여 사용 => 데이터를 순차적으로 탐색             하지 않고 즉시 위치를 찾을 수 있다.
        3) 해시 충돌, 해시 함수는 입력되는 데이터 > 출력되는 해시코드의 범위 => 서로 다른 두             데이터가 동일한 해시코드를 가짐 => 충돌을 최소화 / 충돌 발생시 해결하는 로직이                자료 구조의 핵심

## < 가상화 >
- 가상화 : 물리적 하드웨어 리소스(CPU,메모리, 저장장치 등)를 추상화 -> 하나의 물리적 시스템 위에서 열개의 독립된 가상 환경을 만들어내는 기술
- 가상 머신 : 하이퍼바이저 소프트웨어를 통해 생성된 소프트웨어 기반의 가상 컴퓨터, 각자 고유한 운영 체제와 애플리케이션을 독립적으로 실행
- 하이퍼바이저: 물리적 하드웨어와 가상 머신 사이를 연결하고 리소스를 분할/ 할당하는 핵심 프로그램

- (장점)
  - 비용 절감: 여러대의 물리적 서버를 1대 또는 소수의 서버로 통합 -> 하드웨어 비용, 유지보수비, 전기/냉각 비용을 줄임
  - 자원 효율성: 낭비되는 유휴 하드웨어 자원 최소화, 가상 머신에 리소스를 탄력적으로 할당
  - 격리 및 보안: 각 가상 머신을 파일 형태로 다루는게 가능 -> 복제/ 이전/ 백업 및 재해 복구에 매우 용이함
- (주요 가상화 기술 종류)
  - 서버 가상화: 하나의 물리적 서버를 분할 -> 여러대의 가상 서버처럼 활요하는 기술
    
  - 데스크톱 가상화(VDI): 사용자들의 데스크톱 환경을 중앙 서버에 가상화 -> 개인 단말기(PC,태블릿 등)에서 원격으로 접속해 자업 가능 하게하는 기술
    
  - 네트워크 및 스토리지 가상화: 물리적 네트워크 장치/ 저장장치를 소프트웨어로 추상화하여 통합 및 제어하는 기술
